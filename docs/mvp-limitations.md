# MVP Limitations

Этот список фиксирует осознанные ограничения первой версии, чтобы не путать их с багами.

## Gameplay

- Подбор соперников, вызовы, пошаговый бой, polling-состояние и replay уже вынесены в отдельный flow. Live-обновление работает через периодический HTTP-запрос, без WebSocket/push-канала.
- Навыки работают ограниченно: часть эффектов уже участвует в боевой формуле, часть пока заложена как структура для расширения.
- Расходники можно применять из инвентаря, но временные эффекты на несколько боев пока не реализованы.
- Предметы не имеют веса.
- Нет крафта, торговли между игроками и аукциона.
- Нет восстановления HP по времени.
- Нет отдельной PvE-кампании.
- Нет рейтинговых сезонов и таблицы лидеров.

## Economy

- Токены начисляются только через боевые награды.
- Магазин использует статичный каталог из сидеров и админки.
- Уникальные предметы ограничены владением одного экземпляра у игрока.
- Баланс наград и цен считается стартовым и требует закрытого теста.

## Bots

- Боты нужны для заполнения арены при малом количестве игроков.
- Боты используют простую автотактику по стилю профиля, без полноценного AI и анализа долгой истории боя.
- Боты генерируются из профилей и стартового каталога, без ручной истории развития.

## Interface

- Интерфейс ориентирован на MVP и рабочие сценарии, а не на финальную визуальную полировку.
- Мобильная верстка должна быть рабочей, но расширенная адаптация под маленькие экраны еще впереди.
- Админка использует стандартные возможности Filament без кастомной аналитической панели.

## Operations

- Нет CI/CD pipeline в репозитории.
- Нет автоматического production deploy из GitHub.
- Нет автоматических резервных копий БД в коде проекта.
- Нет встроенного мониторинга ошибок.
- Нет seed-защиты от ручного удаления части связанных данных, кроме обычной повторной генерации стартовых ботов при отсутствии их существ.

## Security

- Email verification не включен.
- Восстановление пароля не включено.
- Двухфакторная авторизация не включена.
- Ограничение частоты запросов используется только стандартное, отдельные игровые action-rate limits еще не настроены.
